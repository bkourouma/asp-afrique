❌ ERREUR 1: TURBO BLOQUÉ
═════════════════════════════════════════════════════════════════════════════

Symptôme:
  $ pnpm dev
  > turbo run dev --parallel
  • Running dev
  • Remote caching disabled
  (Bloqué indéfiniment)

Cause:
  Le script "dev" utilisait "turbo run dev --parallel"
  Turbo essayait de lancer le script "dev" sur tous les packages:
  - ✅ apps/web (a un script "dev")
  - ❌ apps/api (N'A PAS de script "dev")
  - ❌ packages/db (N'A PAS de script "dev")
  
  Turbo attendait que TOUS les packages aient un script "dev"

Solution:
  Changé le script "dev" dans package.json (racine):
  
  AVANT:
  "dev": "turbo run dev --parallel"
  
  APRÈS:
  "dev": "cd apps/web && pnpm dev"

Résultat:
  ✅ L'application démarre correctement
  ✅ Pas de blocage Turbo


❌ ERREUR 2: TYPESCRIPT - 'searchParams' IS POSSIBLY 'NULL'
═════════════════════════════════════════════════════════════════════════════

Symptôme:
  TypeScript Error: 'searchParams' is possibly 'null'
  Fichier: apps/web/src/app/(auth)/login/page.tsx
  Ligne: 27

Cause:
  Le hook useSearchParams() de Next.js peut retourner null
  dans certaines conditions (SSR, edge cases, etc.)

Solution:
  Ajouté optional chaining (?.) pour accéder à searchParams:
  
  AVANT:
  const next = searchParams.get('next') || '/admin'
  
  APRÈS:
  const next = searchParams?.get('next') || '/admin'

Résultat:
  ✅ Erreur TypeScript résolue
  ✅ Code sécurisé contre les valeurs null
